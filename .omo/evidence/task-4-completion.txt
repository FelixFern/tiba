================================================================================
MMKV PERSISTENCE IMPLEMENTATION
================================================================================

Keys Used:
  'tiba_destination'       - Persisted Station object (JSON)
  'tiba_alarm_threshold'   - Persisted alarm threshold (number)

Flow Diagram:
  
  APP STARTUP:
    useTibaStore()
      ↓
    Zustand create() initializes
      ↓
    loadPersistedState() called
      ↓
    MMKV.getString('tiba_destination') → parse JSON → destination state
    MMKV.getNumber('tiba_alarm_threshold') → use or default to 3
      ↓
    Store ready with persisted values

  USER ACTION - SET DESTINATION:
    store.setDestination(station)
      ↓
    Zustand set() updates state
      ↓
    storage.set('tiba_destination', JSON.stringify(station))
      ↓
    MMKV persists JSON data to device storage
      ↓
    Data survives app restart

  USER ACTION - CLEAR DESTINATION:
    store.setDestination(null)
      ↓
    Zustand set() clears state
      ↓
    storage.remove('tiba_destination')
      ↓
    MMKV key deleted from storage

  const setAlarmThreshold = useTibaStore(s => s.setAlarmThreshold);

  const handleSelectStation = (station) => {

  const handleSetAlarm = (num) => {
